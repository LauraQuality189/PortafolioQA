# Backend Testing Project – Restful Booker API

## Project Overview
This project is part of a QA portfolio focused on backend testing.
It uses a public API to simulate real-world API testing scenarios, including requirement analysis, test design, manual testing, and automation.

The goal is to demonstrate QA skills across API validation, test documentation, and automation fundamentals.

---

## System Under Test
The project uses the following API:

Restful Booker API  
https://restful-booker.herokuapp.com/

---

## Testing Scope

This project covers:

- Requirement analysis
- Test case design
- Manual API testing
- API test automation (basic level)
- Regression testing preparation
- Test documentation

---

## Tools Used

- Postman (manual API testing)
- Python
- Pytest (test automation framework)
- Requests library (API calls)
- GitHub (version control)
- Markdown (test documentation)

Optional (simulation / learning context):
- Jira (test case & bug tracking simulation)
- Confluence (documentation structure reference)

## Project Structure

- requirements → API analysis and functional understanding
- test-design → Test cases and scenarios
- api-tests → Manual + automated API tests
